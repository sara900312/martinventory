# 🧴 دليل إعداد نظام التوصيات الجديد
## New Recommendations System Setup Guide

### Overview
نظام التوصيات الجديد يساعد العملاء على إيجاد المنتجات المناسبة لمشاكل بشرتهم. يعمل عن طريق ربط المنتجات بمشاكل البشرة وأنواع الروتين.

---

## 📋 Step 1: Verify Skin Problems Setup

### في قاعدة البيانات:
تأكد من وجود جدول `skin_problems` يحتوي على:
- `id` - معرّف فريد
- `name` - اسم المشكلة بالإنجليزية (مثال: "acne")
- `name_ar` - اسم المشكلة بالعربية (مثال: "حب الشباب")
- `description` - وصف المشكلة
- `emoji` أو `icon` - رمز للمشكلة (مثال: "🌋")

### مثال على المشاكل المدعومة:
```
acne (حب الشباب) - 🌋
dryness (جفاف) - 🏜️
oiliness (دهنية) - 💧
sensitivity (حساسية) - 🔴
aging (الشيخوخة) - 👵
wrinkles (تجاعيد) - 〰️
```

---

## 🛍️ Step 2: Add Products to Recommendations

### في صفحة المخزن (Inventory Page):

#### أ) اختر الفئة المدعومة:
المنتجات يجب أن تكون في الفئات التالية:
- **skincare** (العناية بالبشرة)
- **hair_care** (العناية بالشعر)

> ⚠️ **ملاحظة**: الفئات الأخرى لن تعرض خيارات الروتين والمشاكل

#### ب) ملء البيانات الأساسية:
```
✓ اسم المنتج (Product Name)
✓ السعر (Price)
✓ الكمية (Stock)
✓ الوصف (Description)
✓ الصورة الرئيسية (Main Image)
```

#### ج) اختار مشاكل البشرة المتعلقة:
```
في قسم "🧴 Skin & Routine Settings"
- شاهد جميع مشاكل البشرة المتاحة
- اختار كل المشاكل التي يعالجها المنتج
- مثال: منتج يعالج حب الشباب والدهنية؟
  ✓ اختار كل من: acne, oiliness
```

#### د) اختار نوع الروتين:
```
ثلاثة خيارات متاحة:

🌅 صباحي (Morning)
   - منتجات خفيفة وسريعة الامتصاص
   - مثال: واقي الشمس، تونر، مرطب خفيف

🌙 ليلي (Night)
   - منتجات ثقيلة وعميقة
   - مثال: مصل مكثف، ماسك ليلي، كريم غني

✨ خاص (Special)
   - علاجات متخصصة
   - مثال: مقشر قوي، مصل علاجي، قناع متقدم
```

#### هـ) البيانات الإضافية الاختيارية:
```
- الوصف القصير (Short Description - أهم!)
  • بالعربية: "يعالج حب الشباب ويقلل الدهون"
  • بالإنجليزية: "Treats acne and reduces oil"
  
- التاجز (Tags):
  • اختار من: vegan, hypoallergenic, organic, etc.
  • مثال: منتج حاصل على شهادة كروتي فري
    ✓ cruelty-free
```

#### و) نشر المنتج:
```
يجب أن تكون:
✓ Published = YES
✓ Stock > 0 (إن أمكن)
```

## 🎯 Step 3: Understanding the Data Flow

### عندما يضيف المستخدم منتج:

```
1. ملء النموذج في المخزن
   └─ اختيار مشاكل البشرة والروتين
   
2. حفظ البيانات في قاعدة البيانات
   └─ يتم حفظ أسماء المشاكل كمصفوفة
   └─ مثال: ["acne", "oiliness"]
   
3. نشر المنتج
   └─ المنتج يصبح مرئياً في الأداة
```

### عندما يستخدم العميل الأداة:

```
1. يدخل المستخدم إلى /recommendations
   
2. اختيار مشكلة البشرة
   └─ يتم البحث عن جميع المنتجات مع هذه المشكلة
   
3. اختيار نوع الروتين
   └─ يتم تصفية المنتجات حسب الروتين
   
4. عرض النتائج
   └─ تظهر جميع المنتجات المطابقة
```

## 🔧 Step 4: Troubleshooting

### المشكلة: لا تظهر مشاكل البشرة في النموذج

**الحل:**
1. تأكد أن الفئة = skincare أو hair_care
2. إعادة تحميل الصفحة
3. التحقق من قاعدة البيانات أن جدول skin_problems فيه بيانات

### المشكلة: المنتج لا يظهر في التوصيات

**الحل:**
1. تأكد أن المنتج منشور (published = true)
2. تأكد أن المنتج له stock > 0
3. تأكد أن اخترت مشاكل بشرة وروتين
4. الانتظار دقيقة واحدة (caching)
5. إعادة تحميل صفحة التوصيات

### المشكلة: عدم ظهور الروتين كخيار

**الحل:**
1. تأكد أن الفئة = skincare أو hair_care
2. اختر على الأقل مشكلة بشرة واحدة أولاً
3. الروتين سيظهر بعد اختيار المشكلة
