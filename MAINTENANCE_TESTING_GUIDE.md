# دليل اختبار نظام الصيانة

## 🧪 خطوات الاختبار الأساسية

### الاختبار 1: التحقق من قاعدة البيانات

**الخطوة 1.1:** فتح Supabase
- اذهب إلى [Supabase Dashboard](https://supabase.com)
- اختر مشروعك
- افتح SQL Editor

**الخطوة 1.2:** تشغيل الأوامر
```sql
-- تحقق من وجود الجدول
SELECT * FROM maintenance;

-- أضف سجل اختبار
INSERT INTO maintenance (
  section_name,
  is_active,
  start_time,
  end_time,
  reason
) VALUES (
  'products',
  true,
  now(),
  now() + interval '1 hour',
  'اختبار النظام'
);

-- تحقق من البيانات
SELECT * FROM maintenance WHERE section_name = 'products';
```

### الاختبار 2: التحقق من صفحة الإدارة

**الخطوة 2.1:** الوصول لصفحة الإدارة
- اذهب إلى `/inventory`
- انقر على "إعدادات المخزن"
- اختر "إدارة الصيانة"
- يجب أن ترى: "صفحة إدارة الصيانة"

**الخطوة 2.2:** اختبر إضافة فترة صيانة
1. انقر "فترة صيانة جديدة"
2. اختر "المنتجات" كقسم
3. أدخل السبب: "اختبار شامل"
4. ضع وقت البدء: الآن (now)
5. ضع وقت الانتهاء: بعد 30 دقيقة
6. انقر "إضافة"
7. يجب أن تظهر الرسالة: "تمت الإضافة بنجاح"

### الاختبار 3: التحقق من العداد الزمني

**الخطوة 3.1:** فتح الصفحة التي تستخدم الصيانة
- إذا أضفت الصيانة للمنتجات، اذهب لصفحة المنتجات
- يجب أن تشاهد تنبيهاً برتقالياً يقول:
  ```
  ⚠️ قسم المنتجات تحت الصيانة
  السبب: اختبار شامل
  الوقت المتبقي: 29 د 45 ث
  ```

**الخطوة 3.2:** مراقبة التحديث
- راقب العداد - يجب أن ينقص برقم واحد كل ثانية
- يجب أن يعمل بسلاسة بدون تأخير
- التغيير يجب أن يكون فوري

### الاختبار 4: التحقق من تعطيل الأزرار

**الخطوة 4.1:** اختبر الزر المعطل
- زر "عرض المنتجات" (أو ما يعادله) يجب أن يكون معطلاً
- الزر يجب أن يظهر بلون رمادي
- النقر عليه يجب ألا يحدث أي شيء

**الخطوة 4.2:** اختبر الرسالة
```
الزر يجب أن يكون:
✅ معطل (disabled)
✅ لون رمادي
✅ لا يستجيب للنقر
```

### الاختبار 5: انتهاء الصيانة التلقائي

**الخطوة 5.1:** انتظر انتهاء الصيانة
- إذا أضفت صيانة لمدة 1-2 دقيقة، انتظر
- يجب أن يختفي التنبيه تلقائياً
- يجب أن يتم تفعيل الزر تلقائياً

**الخطوة 5.2:** اختبر مرة أخرى
- انقر الزر - يجب أن يعمل الآن
- لا تحتاج لتحديث الصفحة

### الاختبار 6: تعديل الصيانة

**الخطوة 6.1:** اذهب لصفحة الإدارة
- `/maintenance-management`

**الخطوة 6.2:** اختبر التعديل
1. ابحث عن الصيانة التي أضفتها
2. انقر زر التعديل (قلم)
3. غيّر السبب إلى: "صيانة محدّثة"
4. غيّر وقت الانتهاء
5. انقر "تحديث"
6. يجب أن تظهر رسالة: "تم التحديث بنجاح"

**الخطوة 6.3:** تحقق من التحديث
- اذهب لصفحة المنتجات
- يجب أن تشاهد السبب الجديد
- العداد يجب أن يتحدث للوقت الجديد

### الاختبار 7: حذف الصيانة

**الخطوة 7.1:** اذهب لصفحة الإدارة

**الخطوة 7.2:** اختبر الحذف
1. ابحث عن الصيانة
2. انقر زر الحذف (سلة المهملات)
3. تأكيد الحذف
4. يجب أن تختفي من القائمة

**الخطوة 7.3:** تحقق من التأثير
- اذهب لصفحة المنتجات
- يجب أن يختفي التنبيه
- يجب أن يتم تفعيل الزر

## 🔬 اختبارات متقدمة

### اختبار 8: صيانات متعددة في نفس الوقت

**الخطوة 8.1:** أضف صيانات متعددة
```
- products: 1 ساعة
- categories: 30 دقيقة
- deliveries: 2 ساعة
```

**الخطوة 8.2:** تحقق من التأثير
- كل قسم يجب أن يكون معطلاً بشكل مستقل
- العدادات يجب أن تعمل بشكل منفصل
- الأزرار يجب أن تتعطل بناءً على حالتها فقط

### اختبار 9: اختبار الأداء

**الخطوة 9.1:** راقب الأداء
- افتح DevTools (F12)
- اذهب لتبويب Console
- تحقق من عدم وجود أخطاء

**الخطوة 9.2:** تحقق من الذاكرة
- العداد يجب ألا يسبب تسريب ذاكرة
- لا يجب أن يزيد استهلاك الذاكرة مع الوقت

### اختبار 10: اختبار الأجهزة المختلفة

**الخطوة 10.1:** اختبر على الهاتف الذكي
- شغّل التطبيق على هاتف ذكي
- تحقق من التصميم المتجاوب
- العداد يجب أن يكون واضحاً

**الخطوة 10.2:** اختبر على الأجهزة اللوحية
- تطبيق نفس الخطوات
- تحقق من التخطيط

## 🧹 حالات الاختبار الخاصة

### حالة 1: لا توجد صيانة

**التوقع:**
- لا يظهر تنبيه
- الأزرار تكون مفعلة
- لا يتم استدعاء أي APIs زائدة

**الاختبار:**
```javascript
maintenance.isUnderMaintenance === false
maintenance.reason === null
maintenance.timeRemaining === null
```

### حالة 2: الصيانة نشطة

**التوقع:**
- تنبيه برتقالي يظهر
- عداد صحيح
- الأزرار معطلة

**الاختبار:**
```javascript
maintenance.isUnderMaintenance === true
maintenance.reason === "السبب"
maintenance.formattedTime === "2 س 30 د"
```

### حالة 3: الصيانة قاربت الانتهاء

**التوقع:**
- العداد يظهر أقل من دقيقة واحدة
- الألوان قد تكون أحمر للتحذير

**الاختبار:**
```javascript
maintenance.timeRemaining < 60000 // أقل من دقيقة
```

### حالة 4: خطأ في جلب البيانات

**التوقع:**
- رسالة خطأ واضحة
- زر "حاول مجددا"
- لا يتعطل التطبيق

**الاختبار:**
```javascript
maintenance.error !== null
maintenance.isLoading === false
```

## 🐛 استكشاف الأخطاء الشائعة

### خطأ 1: "لا يمكن قراءة الخاصية 'isUnderMaintenance'"

**السبب:** النيء لم تحصل على البيانات بعد
**الحل:**
```jsx
if (maintenance.isLoading) return <div>جاري التحميل...</div>;
if (maintenance.error) return <div>خطأ: {maintenance.error}</div>;
```

### خطأ 2: العداد لا يتحدث

**السبب:** الـ hook غير متصل صحيحة
**الحل:**
```jsx
useEffect(() => {
  const interval = setInterval(() => {
    // تحديث العداد
  }, 1000);
  return () => clearInterval(interval);
}, [maintenance.endTime]);
```

### خطأ 3: الأزرار لا تتعطل

**السبب:** الخاصية `disabled` غير صحيحة
**الحل:**
```jsx
<button disabled={maintenance.isUnderMaintenance}>
  الزر
</button>
```

### خطأ 4: الصيانة لا تنتهي

**السبب:** `end_time` في المستقبل البعيد جداً
**الحل:**
- تحقق من `end_time` في قاعدة البيانات
- غيّره إلى وقت أقرب للاختبار
