# نظام تكرار عرض الإعلانات المنبثقة
## Popup Display Frequency System

تحسين شامل لميزة تكرار العرض مع ثلاث خيارات مرنة للتحكم في عدد مرات عرض الإعلانات.

---

## 🎯 الخيارات الثلاثة

### 1. **عرض في كل مرة** (Always) - بدون قيود
- ✅ يظهر الإعلان **في كل زيارة** للموقع
- ✅ بدون أي قيود زمنية أو عددية
- 🎯 **الاستخدام المثالي**: عروض عاجلة، إعلانات مهمة، تحديثات حرجة
- 📊 **مثال**: إعلان عن خصم 50% ينتهي اليوم

### 2. **عرض مرة واحدة فقط** (Once)
- ✅ يظهر الإعلان **مرة واحدة فقط** في المتصفح
- ✅ بعد إغلاق الإعلان، لن يظهر مرة أخرى إلا عند مسح بيانات المتصفح
- 🎯 **الاستخدام المثالي**: الترحيب بالزائرين الجدد، تعريف المنتجات الجديدة
- 📊 **مثال**: رسالة ترحيب "أهلاً وسهلاً بك في متجرنا الجديد"

### 3. **عرض كل X مرات** (Every X Visits)
- ✅ يظهر الإعلان **كل عدد معين من الزيارات**
- ✅ مثلاً: عرضه كل 5 زيارات للعملاء المتكررين
- 🎯 **الاستخدام المثالي**: إعلانات متكررة، تذكيرات موسمية، عروض منتظمة
- 📊 **مثال**: عرض إعلان الاشتراك في النشرة البريدية كل 10 زيارات

---

## 🔧 كيفية الاستخدام

### الخطوة 1: الدخول لقسم الإعلانات
```
1. انتقل إلى صفحة الإدارة
2. اختر "إدارة الإعلانات" من القائمة
3. انقر على "إنشاء إعلان جديد" أو عدل إعلان موجود
```

### الخطوة 2: اختيار تكرار العرض
```
في نموذج الإعلان، ابحث عن قسم "تكرار العرض"
```

#### **خيار A: عرض في كل مرة**
```
□ انقر على الخيار "عرض في كل مرة"
□ لا توجد إعدادات إضافية
□ الإعلان سيظهر في كل زيارة
```

#### **خيار B: عرض مرة واحدة فقط**
```
□ انقر على الخيار "عرض مرة واحدة فقط"
□ لا توجد إعدادات إضافية
□ الإعلان سيظهر مرة واحدة فقط
```

#### **خيار C: عرض كل X مرات**
```
□ انقر على الخيار "عرض كل X مرات"
□ أدخل عدد الزيارات (مثلاً: 5)
□ الإعلان سيظهر كل 5 زيارات
```

### الخطوة 3: حفظ وتفعيل
```
1. أكمل ملء بيانات الإعلان الأخرى
2. انقر على "إنشاء الإعلان" أو "تحديث الإعلان"
3. فعّل الإعلان بالضغط على "نشط"
```

## 💾 كيف يعمل التخزين

### localStorage (متصفح العميل)
البيانات تُحفظ محلياً في متصفح كل عميل:

```javascript
// البيانات المحفوظة لكل إعلان
{
  popupId: "popup-123",
  type: "interval",           // 'always', 'once', 'interval'
  interval: 5,                // عدد الزيارات بين العروض
  firstShownAt: "2024-01-15T10:30:00.000Z",
  visitCount: 15,             // عدد الزيارات الكلي
  lastShownAt: "2024-01-15T11:45:00.000Z",
  views: [                    // جميع أوقات العرض
    "2024-01-15T10:30:00.000Z",
    "2024-01-15T10:35:00.000Z",
    "2024-01-15T10:40:00.000Z"
  ]
}
```

## 🧪 اختبار النظام

### 1. اختبار "عرض في كل مرة"
```
1. أنشئ إعلان واختر "عرض في كل مرة"
2. فعّل الإعلان
3. افتح الموقع في متصفح
4. الإعلان يجب أن يظهر
5. أغلق الإعلان
6. أعد تحميل الصفحة
7. الإعلان يجب أن يظهر مرة أخرى ✅
```

### 2. اختبار "عرض مرة واحدة فقط"
```
1. أنشئ إعلان واختر "عرض مرة واحدة فقط"
2. فعّل الإعلان
3. افتح الموقع في متصفح
4. الإعلان يجب أن يظهر
5. أغلق الإعلان
6. أعد تحميل الصفحة
7. الإعلان يجب ألا يظهر ✅
8. امسح بيانات المتصفح (localStorage)
9. أعد تحميل الصفحة
10. الإعلان يجب أن يظهر مرة أخرى ✅
```

### 3. اختبار "عرض كل X مرات" (مثلاً كل 3 زيارات)
```
1. أنشئ إعلان واختر "عرض كل X مرات" وأدخل 3
2. فعّل الإعلان
3. افتح الموقع (الزيارة 1)
4. الإعلان يجب أن يظهر ✅
5. أغلق الإعلان
6. أعد تحميل الصفحة (الزيارة 2)
7. الإعلان يجب ألا يظهر
8. أعد تحميل الصفحة (الزيارة 3)
9. الإعلان يجب ألا يظهر
10. أعد تحميل الصفحة (الزيارة 4)
11. الإعلان يجب أن يظهر ✅ (لأنه مر 3 زيارات منذ آخر عرض)
```

## 🐛 استكشاف الأخطاء

### المشكلة: الإعلان لا يظهر على الإطلاق

**الحل:**
1. تأكد أن `is_active = true`
2. تأكد أن `start_date` في الماضي
3. تأكد أن `end_date` في المستقبل أو فارغ
4. افتح Developer Tools → Console وتحقق من الأخطاء
5. اختبر باستخدام وضع المعاينة (Preview Mode)

### المشكلة: الإعلان يظهر دائماً حتى لو اخترت "عرض مرة واحدة"

**الحل:**
1. افتح Developer Tools → Application → Local Storage
2. ابحث عن `popup_frequency_<popup-id>`
3. امسح هذا الـ key
4. أعد تحميل الصفحة
5. الإعلان يجب أن يظهر مرة واحدة فقط

### المشكلة: الإعلان لا يظهر بالتكرار الصحيح

**الحل:**
1. تحقق من قيمة `frequency_interval` في قاعدة البيانات
2. تأكد أنها تُحفظ بشكل صحيح:
   ```sql
   SELECT id, title, display_frequency, frequency_interval 
   FROM popup_hero 
   WHERE id = '<popup-id>';
   ```
3. امسح localStorage وأعد الاختبار

## 🔄 تغيير الإعدادات

### تغيير التكرار بعد النشر

```
1. افتح الإعلان من قائمة الإعلانات
2. اختر تكرار عرض جديد
3. انقر "تحديث الإعلان"
4. العملاء الجدد سيرون الإعدادات الجديدة
```

### إعادة تعيين للعملاء الحاليين

لإجبار العملاء على رؤية الإعلان مرة أخرى (مفيد عند تغيير الإعدادات):

**الخيار 1:** امسح localStorage يدوياً في متصفحك
```javascript
// اختبار في Developer Tools Console:
localStorage.clear(); // امسح جميع البيانات
```

**الخيار 2:** عدّل معرّف الإعلان في قاعدة البيانات
- هذا يجعل النظام يعامله كإعلان جديد تماماً

---

## 🗄️ متطلبات قاعدة البيانات

### الأعمدة المطلوبة في جدول `popup_hero`:

```sql
- id (UUID, Primary Key)
- title (VARCHAR, 255 chars)
- description (TEXT)
- display_frequency (VARCHAR, 50 chars) - 'always', 'once', 'interval'
- frequency_interval (INTEGER) - عدد الزيارات للمود 'interval'
- is_active (BOOLEAN)
- start_date (TIMESTAMP)
- end_date (TIMESTAMP, nullable)
- image_url (TEXT)
- video_url (TEXT)
- ... (الأعمدة الأخرى)
```

### تشغيل الترقية:

```sql
-- شغّل هذا في Supabase SQL Editor:
ALTER TABLE public.popup_hero 
ADD COLUMN IF NOT EXISTS frequency_interval INTEGER DEFAULT 5;
```

## 🔐 الأمان والخصوصية

✅ **البيانات تُحفظ محلياً في متصفح العميل فقط**
- لا تُرسل البيانات إلى السيرفر
- كل عميل له بيانات منفصلة
- البيانات تُحفظ حتى يمسح العميل localStorage

✅ **لا يتم تتبع هوية العميل**
- النظام يعتمد على localStorage فقط
- لا يتم حفظ معلومات شخصية

⚠️ **ملاحظة مهمة:**
- إذا استخدم العميل متصفحاً مختلفاً، سيكون له سجل منفصل
- إذا مسح العميل بيانات المتصفح، سيُعاد تعيين جميع الإعلانات
